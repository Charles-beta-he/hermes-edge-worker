## 🎯 核心组件

### 1. 多站点管理器 (multi_site_manager.py)

**功能**：
- 站点注册和注销
- 心跳检测
- 健康检查
- 指标统计

**API端点**：
- `POST /sites/register`: 注册站点
- `POST /sites/heartbeat`: 心跳上报
- `GET /sites`: 获取所有站点
- `GET /sites/available`: 获取可用站点
- `GET /metrics`: 获取指标
- `GET /health`: 健康检查

**使用示例**：
```python
from multi_site_manager import MultiSiteManager

# 创建管理器
manager = MultiSiteManager()

# 注册站点
manager.register_site("site-001", {
    "name": "MacBook-Pro-1",
    "ip": "192.168.31.71",
    "port": 9001
})

# 获取可用站点
site = manager.get_available_site()

# 获取指标
metrics = manager.get_metrics()
```

### 2. 站点注册器 (site_registrar.py)

**功能**：
- 注册到主节点
- 心跳上报
- 系统指标收集

**使用示例**：
```python
from site_registrar import SiteRegistrar

# 创建注册器
registrar = SiteRegistrar(
    brain_url="http://192.168.31.71:9009",
    site_id="site-002",
    site_info={
        "name": "MacBook-Pro-2",
        "ip": "192.168.31.130",
        "port": 9002
    }
)

# 注册
registrar.register()

# 启动心跳
registrar.start_heartbeat()
```

### 3. 负载均衡器

**策略**：
- 轮询策略：依次分配任务
- 加权策略：根据站点性能加权
- 最小连接数策略：分配给连接数最少的站点
- 性能策略：根据任务类型和站点性能选择

### 4. 故障转移管理器

**功能**：
- 故障检测：检测站点故障
- 故障转移：将任务转移到其他站点
- 任务重分配：重新分配失败的任务
